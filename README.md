Graph Service (FastAPI + UI)

Сервис для генерации и визуализации различных типов графов для планирования маршрутов дронов:
- **Дорожные сети** из OpenStreetMap
- **Сеточные графы** для свободного пространства (поля, парки)
- **Delaunay триангуляция** для адаптивного покрытия
- **Объединенные графы** (дороги + свободное пространство)

## Установка

1) Создайте и активируйте виртуальное окружение (Windows PowerShell):
```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

2) Установите зависимости:
```powershell
pip install -r requirements.txt
```

**Важно:** Если возникнет ошибка с numpy 2.x, убедитесь что установлен numpy<2.0.0:
```powershell
pip install "numpy<2.0.0" --force-reinstall
```

## Запуск

```powershell
python app.py
```
или через uvicorn:
```powershell
uvicorn app:app --reload --host 0.0.0.0 --port 8000
```

Откройте `http://localhost:8000` в браузере.

## API Эндпоинты

### GET /api/ping
Проверка доступности сервиса

### GET /api/city
Загрузка данных города из OpenStreetMap (с кешированием)

**Параметры:**
- `city` (обязательный) — название города (например: "Volgograd, Russia")
- `network_type` — тип дорожной сети: drive, walk, bike, all, all_private (по умолчанию: drive)
- `simplify` — упрощать ли граф (по умолчанию: true)

**Пример:**
```
GET /api/city?city=Volgograd&network_type=all&simplify=true
```

### GET /api/graph
Генерация графа нужного типа с GeoJSON для визуализации

**Параметры:**
- `city` (обязательный) — название города
- `network_type` — тип дорожной сети (по умолчанию: drive)
- `simplify` — упрощение дорожного графа (по умолчанию: true)
- `graph_type` — тип графа:
  - `road` — только дорожная сеть из OSM
  - `grid` — сеточный граф для свободного пространства
  - `delaunay` — триангуляция Делоне
  - `merged` — объединенный граф (дороги + сетка)
- `grid_spacing` — размер ячейки сетки в градусах (по умолчанию: 0.001 ≈ 100м)
- `connect_diagonal` — соединять диагональные узлы в сетке (по умолчанию: true)

**Примеры:**

Сеточный граф:
```
GET /api/graph?city=Volgograd&graph_type=grid&grid_spacing=0.001
```

Объединенный граф:
```
GET /api/graph?city=Volgograd&graph_type=merged&grid_spacing=0.0015
```

## Типы графов

### 1. Дорожный граф (road)
- Загружается из OpenStreetMap
- Подходит для наземного транспорта
- Цвет: синий

### 2. Сеточный граф (grid)
- **Адаптивная сетка** с квадродеревом (Quadtree)
- Плотная сетка в центре города и важных зонах
- Разреженная сетка на окраинах и в полях
- Автоматически определяет важность зон по:
  - Близости к центру города
  - Плотности зданий
  - Размеру зоны
- Покрывает всё свободное пространство
- Исключает здания и препятствия
- Подходит для дронов над открытыми пространствами
- **Результат:** меньше узлов при лучшем покрытии
- Цвет: зеленый

### 3. Delaunay триангуляция (delaunay)
- Адаптивное распределение узлов
- Качественные треугольники
- Хорош для неравномерных территорий

### 4. Объединенный граф (merged)
- Комбинирует дороги и свободное пространство
- Автоматически соединяет близкие узлы
- Три типа рёбер:
  - **Синие** — дороги
  - **Зеленые** — свободное пространство
  - **Оранжевые** — соединения между графами

## Использование UI

1. Укажите город (например: "Volgograd, Russia")
2. Выберите тип дорожной сети (drive/walk/bike/all)
3. Выберите тип графа:
   - **Дороги** — только дорожная сеть
   - **Сеточный граф** — сетка для свободного пространства
   - **Delaunay** — триангуляция
   - **Объединенный** — комбинация дорог и сетки
4. Настройте параметры сетки (для grid/merged):
   - Размер ячейки (чем меньше — тем плотнее граф)
   - Диагонали (8-связность vs 4-связность)
5. Нажмите "Построить"

Граф отобразится на карте с цветовой кодировкой типов рёбер.

## Алгоритмы

### Grid-based (Сеточный) - Адаптивная версия
- **Квадродерево (Quadtree)** для адаптивного разбиения пространства
- Вычисляет важность каждой зоны на основе:
  - Близости к центру города (40% веса)
  - Плотности зданий (40% веса)
  - Размера зоны (20% веса)
- Адаптивный шаг сетки:
  - Важные зоны: плотная сетка (~30м)
  - Неважные зоны: разреженная сетка (~200м)
- Фильтрует точки внутри зданий/препятствий через пространственный индекс
- Соединяет соседние узлы через KD-tree для быстрого поиска
- Поддерживает 4-связность и 8-связность
- **Преимущества:**
  - В 2-5 раз меньше узлов при том же покрытии
  - Быстрее генерация для больших городов
  - Лучшее качество графа в важных зонах

### Delaunay Triangulation
- Размещает случайные точки в свободном пространстве
- Строит триангуляцию Делоне
- Рёбра треугольников становятся рёбрами графа
- Фильтрует рёбра, пересекающие препятствия

### Graph Merging
- Объединяет дорожный и свободный графы
- Находит близкие узлы между графами
- Создает соединяющие рёбра
- Сохраняет метаданные о типе узлов/рёбер

## Кэширование

- **Файловый кэш:** данные городов сохраняются в `cache/`
- **Redis (опционально):** установите `REDIS_URL` для ускорения
- Кэш учитывает: город, тип сети, simplify

## Примечания

- Пакеты `osmnx` и `geopandas` могут требовать системные зависимости (GEOS/GDAL/Proj)
- На Windows рекомендуется использовать готовые wheel-файлы или conda
- Для больших городов генерация графа может занять время
- Размер сетки 0.001° ≈ 100 метров
- Чем меньше размер ячейки, тем больше узлов и медленнее генерация

## Docker

Собрать и запустить локально (только приложение):
```powershell
docker build -t graph_service .
docker run --rm -p 8000:8000 -e REDIS_URL=redis://host.docker.internal:6379/0 graph_service
```

Docker Compose (с Redis):
```powershell
docker compose up --build
```
Затем откройте: `http://localhost:8000`
